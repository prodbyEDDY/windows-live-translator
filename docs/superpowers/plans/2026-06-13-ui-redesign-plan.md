# UI Redesign Plan (multi-agent audit, 2026-06-13)

Generated by a 7-agent audit grounded in the installed Impeccable + taste design skills.

I have enough grounding in the actual token system and component structure. The audit findings are detailed and accurate against the real code. Let me produce the consolidated, implementation-ready redesign plan.

# Live Translator — Unified Redesign Plan

A single design system across all five screens (Wizard, Live, Voice, History, Settings) plus shared chrome. **Implement Section 1 (Shared design system) first** — every per-file fix references those tokens/components. Section 2 is the de-duplicated per-file checklist, ordered so foundation precedes screen work.

The audits agree on five cross-cutting problems that appear in nearly every file. Fixing them once at the system level resolves the majority of individual findings:
1. **Side-stripe accent bars** (Banner, VoiceCard, Sidebar) — banned pattern, fixed once in Banner + a shared dot.
2. **Sub-AA contrast** on muted/tinted/disabled text — fixed once via token changes + a status-chip recipe.
3. **No type scale** — every screen invents one-off px sizes; fixed once with `--text-*` tokens.
4. **Raw Unicode/emoji icons** (♪ 🔊 ⇄ → ◎ ⎘ ▶ ⠿ ⌬) — fixed once by extending Icons.tsx.
5. **Conflated loading/empty/error states** — fixed once with a shared Skeleton + inline error pattern.

## 1. Shared design system

Do these in order. They are the foundation; do not start per-screen work until §1.1–§1.6 land.

### 1.1 Color tokens — fix contrast and complete the ramps (`src/index.css`)

Replace/extend the `@theme` block. Concrete values:

```css
@theme {
  /* Surfaces & ink — cool-neutral paper, not the flagged warm cream */
  --color-paper: #f7f7f8;        /* was #faf9f6 (warm cream/sand AI default) */
  --color-surface: #ffffff;       /* keep pure white for max contrast */
  --color-ink: #1c1917;
  --color-ink-2: #44403c;         /* NEW: secondary ink for labels (~9:1) */
  --color-muted: #6b6560;         /* was #78716c; nudge darker to clear 4.5:1 on paper */
  --color-hairline: #e7e5e4;
  --color-hairline-strong: #d6d3d1; /* NEW: hover borders / scrollbar thumb, replaces ad-hoc stone-300 */

  /* out = cobalt (you) */
  --color-cobalt: #1e5bd7;
  --color-cobalt-tint: #eef3fe;
  --color-cobalt-deep: #173fa0;

  --color-tangerine: #e2620e;
  --color-tangerine-tint: #fdf1e7;
  --color-tangerine-deep: #b34a05;   /* white-on-this ≈ 5.6:1 → use for running "in" chip */

  /* Semantic — base + deep (AA text) + tint (chip/banner ground) */
  --color-ok: #177e5b;
  --color-ok-deep: #0f6446;          /* NEW: AA text on ok-tint */
  --color-ok-tint: #e7f1ec;          /* NEW */
  --color-danger: #c2362b;
  --color-danger-deep: #9e271e;      /* NEW */
```

- `--color-paper` to cool-neutral `#f7f7f8` (chroma ~0) so warmth is a deliberate non-choice; the duotone carries identity. With muted at `#6b6560`, body/placeholder text clears 4.5:1 on paper.
- These token changes alone resolve the contrast findings in Settings, History, Banner, Header (muted-on-paper) without touching those files' markup.

### 1.2 Type-scale + label utilities

Map the new `--text-*` tokens to Tailwind via the `@theme` naming (Tailwind v4 exposes `--text-*` as `text-*` utilities automatically). After this, replace every `text-[Npx]` literal with: `text-code | text-label | text-caption | text-body | text-emphasis | text-h2 | text-h1`. Eliminate `11.5px`, `12.5px`, `13.5px`, `9px`, `10px` body usages — collapse to the nearest token (chips→`text-caption`, micro-labels→`text-label`).

**One label role.** Define a single eyebrow utility and use it nowhere else but section/meter labels:
```
.lt-eyebrow { font: 500 var(--text-label)/1 var(--font-body); text-transform: uppercase; letter-spacing: 0.08em; color: var(--color-ink-2); }
```
Reserve `0.14em` uppercase + Unbounded for **section titles only**. The wordmark moves to a logotype treatment (see Header) and stops sharing the eyebrow grammar.

### 1.3 Kill the side-stripe pattern — rewrite Banner once (`src/components/Banner.tsx`)

This component is rendered by Live, Voice, Wizard, History — fixing it here removes the banned stripe from four screens at once.

- **Delete** the `<span className="absolute left-0 top-0 bottom-0 w-[3px] ...">` bar and the `pl-4`/`overflow-hidden` that compensated for it. Use symmetric `px-3.5 py-2.5`.
- Carry tone via **full tinted hairline border + leading tone icon**:
  ```
  TONE = {
    warn:   { tint:"bg-warn-tint",   border:"border-warn/30",   title:"text-warn-deep",   Icon: IconAlert },
    danger: { tint:"bg-danger-tint", border:"border-danger/30", title:"text-danger-deep", Icon: IconAlert },
    ok:     { tint:"bg-ok-tint",     border:"border-ok/30",     title:"text-ok-deep",     Icon: IconCheck },
    info:   { tint:"bg-cobalt-tint", border:"border-cobalt/30", title:"text-cobalt-deep", Icon: IconInfo },
  }
  ```
  Root: `relative flex items-start gap-2.5 rounded-card border ${border} ${tint} px-3.5 py-2.5`. Render `<c.Icon size={16} className={c.title} aria-hidden />` first.
- Description: `text-caption text-ink-2 leading-snug` (no longer muted-on-tint).
- Title: `text-body font-semibold ${c.title}` using the new `*-deep` tokens (no hardcoded hex).

### 1.4 Shared accent-dot, status-chip, and skeleton primitives

Add to a small `src/components/primitives.tsx` (or extend Icons/existing helpers) so screens stop re-implementing these:

- **`<AccentDot tone="out"|"in" />`** → `<span className="w-2 h-2 rounded-full shrink-0" style={{background: tone==='out' ? 'var(--color-cobalt)' : 'var(--color-tangerine)'}} />`. This is the replacement for every direction side-stripe on cards (VoiceCard, Sidebar active marker).
- **`<StatusChip tone variant>`** — one verified-contrast recipe for all chips (Header session chip, ApiKeyField key chip, Settings VB-CABLE, Live direction chip idle states):
  - ok: `bg-ok-tint text-ok-deep`
  - warn: `bg-warn-tint text-warn-deep`
  - danger: `bg-danger-tint text-danger-deep`
  - neutral/off: `bg-stone-100 text-ink-2`
  - shared: `inline-flex items-center gap-1.5 h-6 px-2.5 rounded-pill text-caption font-medium`
  - "running/active" filled variant: out→`bg-cobalt text-white`, in→`bg-tangerine-deep text-white` (NOT `bg-tangerine` — that fails AA at 3.51:1).
- **`<Skeleton className>`** → `bg-stone-100 rounded animate-pulse` (animation gated by reduced-motion already via the global rule). Plus shaped variants used by screens: `<SkeletonRow>` (History, h-11), `<SkeletonCard>` (Voice/Settings), `<SkeletonBubble side>` (Live).
- **`useDir(lang)`** helper returning `"rtl"|"ltr"` for ar/he/fa/ur, for the RTL fix.

### 1.5 Icon set — extend and standardize (`src/components/Icons.tsx`)

Add, in the existing 24×24 stroke-1.5 currentColor style, so no screen uses raw glyphs:
`IconAlert` (warn/danger banners), `IconInfo` (info banner), `IconChevron` (History disclosure, separators — rotate via `rtl:rotate-180` / expand state), `IconCopy` (Voice/History copy), `IconPlay` (Voice), `IconTrash` (Live clear, History clear), `IconDesktop` + `IconAppWindow` (Live source mode), `IconExternalLink` (Settings/Wizard "Get key"), `IconInbox`/empty glyph (empty states), `IconLock` (Header locked controls).

Standardize existing: remove the `M4 12v0`/`M21 12v0` zero-length dot hacks in `IconWaveform`; make `IconGrip` consistent (keep filled dots but document, or convert to stroked) — pick filled and note it's intentional. Keep `WaveformGlyph` (legit bespoke brand mark).

### 1.6 Remove flagged global tells (`src/index.css`, `src/App.tsx`)

- **Delete `.grain-overlay`** (feTurbulence) and its usages on `main`/wizard root. A calm professional tool wants flat surfaces; duotone + type carry personality.
- `h-screen` → `h-[100dvh]` on both the wizard wrapper and main shell in `App.tsx`.
- Add `dir` to the document root driven by UI locale, and `hairline-strong` now replaces the `#d6d3d1` literals in the scrollbar rules.
